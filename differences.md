- vue components sind objects, react components sind functions. Der meiste code läuft in vue in der setup function, und daher nur ein mal. React wiederum durchläuft den code bei jedem render. Das heißt einerseits performance-einbußen, sorgt aber auch dafür, dass variablen immer reaktiv aktualisiert werden. In vue hingegen werden werte nur beim parsing des template ausgelesen, oder in computed funktionen.
- react hooks und vue composables verwenden ziemlich das gleiche Konzept.
- react contexts können wie bekannt mit provide/inject nachgebildet werden. Die menge an code ist dabei sogar weniger. React contexts wirken allerdings mehr wie ein teil der component struktur, was sie theoretisch intuitiver macht. Pinia kann bei vue die gleiche intuition geben, wirkt aber für mich aufwendiger als provide/inject ohne erkennbaren vorteil
- react/jsx erlaubt richtigen javascript code in der html struktur, was es deutlich lesbarer und leichter zu programmieren macht und typesafety gibt. for loops sind allerdings ähnlich wenig gut, solang nicht gemapt werden
